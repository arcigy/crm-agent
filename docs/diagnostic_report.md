# ArciGy CRM: Extreme Diagnostic Report

## 1. Executive Summary
The CRM system is architecturally sound in its "Agentic" core (7-layer pipeline) but suffers from critical integration gaps and security risks in its data persistence and Google Workspace layers. The most pressing issues are the **hardcoded Directus tokens** and the **complete lack of background synchronization** for external services.

## 2. Security Audit
| Finding | Severity | Description | Recommendation |
| :--- | :--- | :--- | :--- |
| **Hardcoded Fallback Token** | 🔴 CRITICAL | `src/lib/directus.ts` contains a hardcoded `DIRECTUS_TOKEN`. | Remove fallback and enforce `.env` validation in `src/lib/directus.ts`. |
| **Broad OAuth Scopes** | 🟡 MEDIUM | `src/lib/google.ts` requests `drive`, `gmail.modify`, and `contacts` globally. | Implementation of incremental scoping or per-module scope request. |
| **Exposed Private Networking** | 🟢 LOW | API routes reference `.railway.internal` URLs which fail locally. | Standardize on public URLs for dev and internal for production with proper fallbacks. |

## 3. Data Architecture & Relational Integrity
- **Relational Status**: Confirmed that `projects`, `deals`, and `activities` are separate collections.
- **Relational Issues**: `activities` contains a `contact_id` which is verified to be consistent in samples.
- **Token Integrity**: Found `google_tokens` records missing `refresh_token`.
    - **Impact**: Integration breaks permanently after 1 hour for these users.
    - **Root Cause**: Likely initial OAuth consent without `access_type: offline`.

## 4. Integration Gaps (CRITICAL)
### Pull-Only Sync
Currently, Google Calendar and Gmail data is only fetched when the user visits the page.
- **Problem**: CRM "Leads" are not created automatically when emails arrive.
- **Problem**: Meetings created in external apps don't trigger CRM automations.

## 5. Module-Specific Findings
- **Contacts**: "Push-on-write" sync exists but lacks "Conflict Resolution" (last-write-wins).
- **Calendar**: Timezone handling relies on UTC `toISOString()`, which may cause offset issues for non-UTC users.
- **Agent Pipeline**: The `agent-registry.ts` is over 1600 lines. While robust, it is a maintenance bottleneck.

## 6. Infrastructure Performance
- **N+1 Logic**: Certain API routes fetch from Google and then loop over results to verify in Directus. This will scale poorly for users with >1000 events/emails.
- **Chaining Crons**: The worker chaining (`fetch` calling the next worker) is clever but prone to timeout breaks if one link in the chain fails.
